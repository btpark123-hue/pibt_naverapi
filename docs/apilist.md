# 네이버 오픈API 종류 - NAVER Developers

## 개요

네이버 오픈API는 서비스의 성격과 보안 수준에 따라 크게 **로그인 방식**과 **비로그인 방식**으로 구분됩니다. 각 방식은 인증 절차와 사용할 수 있는 기능이 다르므로, 개발하려는 서비스의 목적에 맞는 API를 선택하는 것이 중요합니다.

## 1. 로그인 방식 오픈 API

네이버 로그인(OAuth 2.0) 인증을 거쳐 사용자의 동의를 얻은 후, **접근 토큰(Access Token)**을 발급받아야 사용할 수 있는 API입니다. 사용자의 개인화된 데이터에 접근하거나 네이버 서비스에 직접적인 액션(글쓰기 등)을 취할 때 사용합니다.

### 주요 서비스

- **네이버 로그인**: 별도의 회원가입 없이 네이버 아이디로 웹/앱 서비스에 로그인할 수 있는 기능을 제공합니다.
- **내 프로필 정보**: 사용자의 별명, 프로필 사진, 이메일, 생일 등의 정보를 가져옵니다.
- **네이버 카페**: 사용자가 가입한 카페 리스트를 확인하거나, 특정 카페에 게시글을 등록합니다.
- **네이버 캘린더**: 사용자의 일정 조회, 등록, 수정, 삭제 기능을 제공합니다.
- **네이버 메모**: 사용자의 메모 데이터를 연동합니다.

## 2. 비로그인 방식 오픈 API

사용자 로그인 과정 없이 애플리케이션에 발급된 **Client ID**와 **Client Secret**만으로 호출 가능한 API입니다. 주로 공개된 정보를 검색하거나 공용 유틸리티 기능을 사용할 때 활용합니다.

### 주요 서비스

- **검색 API**: 블로그, 뉴스, 책, 쇼핑, 이미지, 지식iN 등 네이버의 방대한 검색 결과 데이터.
- **데이터랩(DataLab) API**: 검색어 트렌드 분석 및 쇼핑 카테고리별 클릭 분석 데이터.
- **보안/유틸리티 API**: 캡차(이미지/음성), 단축 URL 생성 API.
- **인공지능(AI) API**: CLOVA Face Recognition(얼굴인식), CLOVA Speech(STT) 등.
- **플러그인**: 네이버 공유하기, 오픈메인 추가 기능 등.

## API 방식 비교 및 선택 가이드

| 비교 항목 | 로그인 방식 오픈 API | 비로그인 방식 오픈 API |
| :--- | :--- | :--- |
| **인증 방식** | OAuth 2.0 (접근 토큰) | Client ID & Client Secret (Header) |
| **사용자 동의** | 명시적 동의 필요 (로그인 창) | 불필요 |
| **주요 목적** | 개인 데이터 연동, 서비스 액션 | 정보 검색, 공용 기능 활용 |
| **구현 난이도** | 중간 (인증 흐름 구현 필요) | 낮음 (단순 HTTP 요청) |

## API 사용 시 공통 사항

### 1. 애플리케이션 등록

모든 API는 [네이버 개발자 센터]에서 애플리케이션을 먼저 등록해야 사용할 수 있습니다. 등록 시 사용할 API 권한을 명확히 선택해야 합니다.

### 2. 호출 한도 (Quota)

- 모든 API에는 일일 호출 한도가 정해져 있습니다.
- 한도는 애플리케이션 단위로 계산되며, 한도 초과 시 당일 API 호출이 중단됩니다.
- 한도 상향이 필요한 경우 제휴 신청을 통해 검토받을 수 있습니다.

### 3. 보안 주의 사항

- **Client Secret**과 **접근 토큰**은 절대로 외부로 노출되어서는 안 됩니다.
- 클라이언트 단(Javascript)에서 직접 호출하기보다는 서버 단(Server-to-Server)에서 호출하여 인증 정보를 보호하는 것이 안전합니다.
