# 검색 API 개요 - NAVER Developers

## 개요

네이버 검색 API는 네이버가 보유하고 있는 방대한 웹 콘텐츠(블로그, 뉴스, 이미지, 쇼핑 등) 데이터를 외부 애플리케이션에서 검색하고 서비스에 활용할 수 있도록 지원하는 RESTful API입니다.

네이버의 핵심 검색 기술과 수집 엔진이 분석한 고품질의 데이터를 실시간으로 가져와 다양한 사용자 경험을 창출할 수 있습니다.

## 검색 API의 분류

네이버 검색 API는 정보의 성격에 따라 다양한 서비스로 세분화되어 있습니다.

### 1. 콘텐츠 및 정보 검색

- **블로그(Blog)**: 최신 블로그 포스트 및 개인 의견 정보 검색.
- **뉴스(News)**: 주요 언론사 및 포털 뉴스의 실시간 기사 수집.
- **책(Book)**: 도서 서지 정보, 가격, 출간일 등 상세 데이터.
- **백과사전(Encyclopedia)**: 검증된 학술 및 전문 지식 정의 검색.
- **지식iN(Kin)**: 사용자 간 질문과 답변(Q&A) 데이터 제공.

### 2. 쇼핑 및 위치 기반 검색

- **쇼핑(Shopping)**: 상품명, 가격비교, 입점 몰 정보 검색.
- **지역(Local)**: 업체 전화번호, 주소, 위치 좌표 등 장소 정보.
- **전문자료(Document)**: 학술 논문 및 전문 연구 자료 검색.

### 3. 멀티미디어 및 분석 도구

- **이미지(Image)**: 원본 고해상도 이미지 및 섬네일 정보 수집.
- **오타변환(Errata)**: 검색어의 오타 교정 및 한/영 자판 보정.
- **성인 검색어 판별**: 키워드의 유해 여부 필터링.

## 공통 인증 규격

검색 API는 **비로그인 방식 오픈 API**로, 단순한 HTTP 요청 헤더에 발급받은 키를 포함하는 것만으로 호출이 가능합니다.

- **인증 헤더**:
  - `X-Naver-Client-Id`: {발급받은 **Client ID**}
  - `X-Naver-Client-Secret`: {발급받은 **Client Secret**}
- **데이터 형식**:
  - 모든 검색 API는 응답 결과로 XML 또는 JSON 형식을 선택할 수 있습니다. (URL 확장자로 구분)

## 공통 요청 파라미터 리스트

대부분의 검색 API는 아래와 같은 공통 파라미터를 사용합니다.

| 매개변수 | 타입 | 공통 요건 | 설명 |
| :--- | :--- | :--- | :--- |
| `query` | String | 필수 | 검색 키워드 (UTF-8 URL 인코딩 필수) |
| `display` | Integer | 선택 | 한 번에 노출될 아이템 수 (기본 10, 최대 100) |
| `start` | Integer | 선택 | 검색 결과의 시작 위치 (최대 1000) |
| `sort` | String | 선택 | 결과의 정렬 기준 (유사도순/날짜순 등) |

## 서비스 이용 한도 (Quota)

- **일일 호출 한도**: 검색 API 전체 통합 총 **25,000건** (일별 기준).
- **한도 상향**: 대규모 트래픽이 예상되는 제휴사의 경우 네이버 개발자 센터를 통해 별도 한도 조절 문의가 가능합니다.
- **초과 시 대응**: 한도 초과 시 429 응답이 오며, 한국 시간 기준 자정에 호출 횟수가 초기화됩니다.

## 보안 및 권장 사항

1. **서버 간 통신(Server-to-Server)**: Client Secret 노출을 막기 위해 브라우저 단보다는 서버 사이드에서 API를 호출할 것을 강력히 권장합니다.
2. **인코딩 처리**: `query` 값에 들어가는 한글이나 특수 문자는 반드시 표준 URL 인코딩 처리를 해야 데이터 깨짐이 없습니다.
3. **태그 처리**: 응답 결과 중 `title`이나 `description`에 포함된 HTML 강조 태그(`<b>`, `</b>`)는 화면 노출 시 사용자 환경에 맞게 적절히 처리해야 합니다.
