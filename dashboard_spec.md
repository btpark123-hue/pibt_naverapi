# 작업지시서: Naver API 통합 트렌드 대시보드

## 1. 프로젝트 개요

- **목적**: 네이버 검색 및 데이터랩 API를 활용하여 특정 키워드의 트렌드를 분석하고 관련 콘텐츠(쇼핑, 블로그, 카페, 뉴스 등)를 한눈에 파악할 수 있는 대시보드 구축.
- **기술 스택**: Python, Streamlit, Plotly, Pandas, Requests.

## 2. 주요 기능 및 요구사항

### A. 키워드 검색 및 트렌드 비교

- 사용자로부터 여러 개의 키워드를 입력받아 네이버 데이터랩(검색어 트렌드) API를 통해 비교 분석.
- **시각화**: Plotly를 사용하여 선형 차트 등으로 구현.

### B. 메뉴 및 탭 구성 (5가지 시각화 및 데이터 표)

- **트렌드 분석 탭**: 일별/월별 검색 추이 (Plotly Line Chart, Area Chart)
- **콘텐츠 분석 탭**: 채널별(블로그/카페/뉴스) 언급량 비교 (Plotly Pie Chart, Bar Chart)
- **사용자 분포 탭**: 성별/기점별 관심도 (Plotly Heatmap)
- **쇼핑 트렌드 탭**: 카테고리별 클릭 추이 (Plotly Scatter Plot)
- **통합 요약 탭**: 주요 지표(KPI) 카드 및 상관관계 분석.

### C. 콘텐츠 채널별 리스트 구현

- **대상**: 네이버 쇼핑, 블로그, 카페, 뉴스.
- **기능**:
  - 결과 목록 출력 (제목, 요약, 링크).
  - 링크 클릭 시 해당 원본 페이지로 이동.
  - 리스트 하단에 **페이징(Pagination)** 기능 추가.

## 3. Naver API 활용 가이드 (docs 폴더 참조)

- `datalab_search_trend.md`: 트렌드 분석용 데이터 수집.
- `shopping.md`, `blog.md`, `kin.md`, `news.md`: 각 채널별 검색 결과 수집.
- `search_overview.md`: 공통 파라미터 및 인증 방식 확인.

## 4. 데이터 관리

- 실시간 API 호출을 기본으로 하되, 성능을 위해 `st.cache_data` 활용.
- 수집된 데이터는 Pandas DataFrame으로 가공하여 시각화에 전달.
